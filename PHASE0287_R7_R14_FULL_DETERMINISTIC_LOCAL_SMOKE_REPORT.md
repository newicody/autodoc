# Phase 0287-r7-r14 — Full deterministic local smoke

## Objective

Close one correlated local proof across the already implemented r7, r11, r12
and r13 boundaries:

```text
Issue fixture
→ three already-fetched GitHub artifact members
→ run assembly and intake
→ correlated research work package
→ explicit operator promote/merge gate
→ existing Scheduler, first native specialist visit
→ existing Scheduler, second native specialist visit
→ SQL analysis authority
→ OpenVINO/E5-384 projection and Qdrant reference recall
→ SQL rehydration and liaison synthesis
→ final artifact envelope
→ Issue + ProjectV2 publication preview
→ exact simulated readback
```

No remote mutation is performed.

## Reuse audit

The phase composes existing public surfaces:

- `github_dual_artifact_run_assembly_0281.py` for three-artifact correlation;
- `correlated_research_work_package_0287.py` for the authoritative request,
  consultative advisory and portable package;
- `source_candidate.py` for the explicit operator decision;
- `native_love_laboratory_collaboration_scheduler_binding_0287.py` and
  `native_love_laboratory_second_specialist_0287.py` for the two separately
  scheduled specialist visits;
- `love_memory_evidence_liaison_synthesis_0287.py` for SQL authority,
  OpenVINO/E5-384, Qdrant recall, evidence mutualization and final envelope;
- `love_final_deliverable_publication_plan_0287.py` for Issue/ProjectV2 preview
  and exact readback verification.

The only new bridge maps an approved correlated work package into the existing
`LoveStudyRequest`, first `SpecialistTaskRequest` and first
`LaboratoryVisitRequest` contracts. It does not create an execution authority.

## Result

`love_full_deterministic_local_smoke_0287.py` exposes one asynchronous
composition function with injected Scheduler, Dispatcher, SQL authority store,
projection port, collection profile, embedder and retrieval executor.

The proof:

- requires exactly the authoritative request, Copilot advisory and correlation
  manifest artifact members;
- requires an explicit `promote` or `merge` operator decision;
- keeps the authoritative request as study title and subject text;
- retains the advisory reference as consultative provenance only;
- submits both visits through the same externally supplied Scheduler;
- persists and rehydrates both analyses through the r12 chain;
- creates the r13 publication preview;
- verifies an exact local Issue comment and ProjectV2 field readback;
- computes a deterministic proof digest from stable domain references, not from
  randomly generated Scheduler event identifiers.

## Authority boundaries

- Scheduler: reused, not created or modified.
- Dispatcher: injected existing registration surface.
- SQL: durable analysis and synthesis authority, exercised locally.
- Qdrant: projection and recall only, exercised through injected ports.
- OpenVINO/E5: explicit 384-dimensional query embedding contract reused.
- EventBus: unchanged.
- ControlProxy: unchanged.
- GitHub Issue mutation: not performed.
- ProjectV2 mutation: not performed.
- Readback: exact simulation only.

## Code-rule review

- existing wheels and contracts were audited before adding the bridge;
- no manager, new Scheduler, queue, registry or parallel orchestrator exists;
- the command and result are immutable typed dataclasses;
- business composition remains in `src/context`, not in a CLI;
- IDs and the proof digest are deterministic SHA-256 derivations;
- network clients and subprocesses are absent;
- DOT remains source-only and contains architecture, not policy text.

## Failure modes

The smoke fails closed when:

- one of the three required artifacts is absent, duplicated or tampered;
- intake or work-package correlation fails;
- the operator action is not `promote` or `merge`;
- a merge lacks its target context;
- the candidate already carries a prior decision;
- return-route or context references do not match the existing laboratory and love-study prefixes;
- either Scheduler visit fails or returns an incoherent receipt;
- SQL base revision is absent;
- projection digest or E5/Qdrant recall is incomplete;
- r12 synthesis is not publication-ready;
- r13 preview is blocked or collides;
- simulated Issue or ProjectV2 readback differs from the approved plan;
- any remote mutation flag is enabled.
