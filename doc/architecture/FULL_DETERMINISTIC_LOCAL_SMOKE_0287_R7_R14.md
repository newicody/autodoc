# Full deterministic local smoke — 0287-r7-r14

## Scope

This phase is the first single proof that crosses the complete local love-study
prototype path. It composes existing modules and injected ports; it is not a new
runtime framework.

## Flow

1. Three byte members representing already-downloaded GitHub Actions artifacts
   enter the run-level assembler.
2. The existing intake validates request, advisory and manifest digests and
   builds a `SourceCandidate` mapping from the authoritative request only.
3. The r7 builder creates the correlated work package.
4. The supplied immutable operator decision promotes or merges the candidate.
5. A deterministic bridge creates a `LoveStudyRequest`, first specialist task
   and first laboratory visit.
6. The supplied Scheduler executes the first visit through the registered
   collaborative native provider.
7. The first analysis artifact is registered; a second visit is prepared and
   separately submitted through the same Scheduler.
8. The r12 composition persists analyses in SQL, projects E5-384 vectors,
   recalls Qdrant references, rehydrates SQL objects and builds liaison synthesis
   plus the final envelope.
9. The r13 planner creates exact Issue and ProjectV2 publication intentions.
10. Local snapshots reproduce the approved body and value; the existing
    verifier confirms exact readback.

## Determinism

Scheduler event IDs remain runtime evidence and may vary. They are deliberately
excluded from the stable proof digest. The digest covers stable package,
candidate, visit, revision, publication-plan and readback references.

## Boundaries

The authoritative request remains the study source. Copilot output remains an
untrusted hint and provenance reference. SQL remains durable authority. Qdrant
contains projection metadata and retrieval references. The Scheduler remains
the only execution authority for both visits. GitHub and ProjectV2 are previewed
but not mutated.
