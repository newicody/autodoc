# 0126 — Operational orientation

0126 adds vector contract indexing contracts without introducing runtime clients.

The operational direction is:

```text
GitHub artifact
-> SQL import
-> ServerOrientation
-> Scheduler-owned deliberation route contracts
-> vector retrieval plan over contracts/context/output/signal collections
-> /dev/shm route demand frames
-> specialist work
-> specialist output indexing descriptor
-> SQL persistence
-> Qdrant projection by role
-> final synthesis candidate retrieval
-> GitHub final artifact exchange later
```

Qdrant is not split per specialist. It is split by collection role. Specialist identity stays in payload/filter metadata.

E5/OpenVINO is used only as the embedding adapter for `query:` and `passage:` text. It does not validate, synthesize, decide, or orchestrate.
