# Copilot advisory v2 board and Issue publication

## Flow

```text
GitHub Actions artifacts
  -> strict correlation and digest validation
  -> publication preview v2
  -> ProjectV2 projection plan v2
       -> Copilot = Terminé
       -> Avis Copilot = four-field bounded summary
       -> date, artifact and cycle
  -> Issue publication plan v1 for advisory-v2 content
       -> deterministic Markdown
       -> distinct v2 marker
       -> create / replay / collision
  -> explicit operator and mutation gates
  -> exact ProjectV2 field and Issue comment readback
```

## Versioning

The existing v1 preview and publication plans are not reinterpreted. The v2
preview, v2 ProjectV2 plan and advisory-v2 Issue plan have distinct schemas.

## Semantic safety

The v2 path never fabricates a route or confidence value. Specialists and the
future laboratory receive the advisory later as consultative context only.
