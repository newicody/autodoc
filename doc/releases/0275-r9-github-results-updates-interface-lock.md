# 0275-r9 — GitHub results/updates interface lock

This phase locks the final human-facing GitHub model:

- one current result card per research;
- research sources represented only by references;
- separate parent and related-result links;
- append-only timestamped updates;
- short server and Copilot summaries on cards;
- full content in the issue panel;
- separate server-action and history views.

Runtime collection and controlled GitHub publication remain deferred.
