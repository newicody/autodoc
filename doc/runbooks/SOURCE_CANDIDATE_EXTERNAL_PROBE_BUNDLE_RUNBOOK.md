# SourceCandidate External Probe Bundle Runbook

This runbook describes the local operator workflow for the SourceCandidate
external probe path.

The workflow remains local-only:

```text
no external service call
no token handling
no remote mutation
no Scheduler execution
```

## Inputs

The operator starts from local artifacts generated by the previous phases:

```text
operator_external_review_report.json
read_only_external_probe_request.json
read_only_external_probe_result.json
```

The expected schemas are:

```text
missipy.source_candidate.operator_external_review_report.v1
missipy.source_candidate.read_only_external_probe_request.v1
missipy.source_candidate.read_only_external_probe_result.v1
```

## Dry-run plan

Use the CLI without `--apply` first:

```bash
python tools/source_candidate_external_probe_bundle_cli.py \
  --output-dir /tmp/source_candidate_external_probe_bundle \
  --operator-review-report /path/to/operator_external_review_report.json \
  --probe-request /path/to/read_only_external_probe_request.json \
  --probe-result /path/to/read_only_external_probe_result.json
```

Expected result:

```text
external probe bundle cli: dry-run
writes_bundle: False
```

## Apply local bundle

Only after the dry-run plan is correct:

```bash
python tools/source_candidate_external_probe_bundle_cli.py \
  --output-dir /tmp/source_candidate_external_probe_bundle \
  --operator-review-report /path/to/operator_external_review_report.json \
  --probe-request /path/to/read_only_external_probe_request.json \
  --probe-result /path/to/read_only_external_probe_result.json \
  --apply
```

Expected bundle:

```text
/tmp/source_candidate_external_probe_bundle/
  manifest.json
  operator_external_review_report.json
  read_only_external_probe_request.json
  read_only_external_probe_result.json
```

## Validate repository state

Run the normal local gates:

```bash
PYTHONPATH=src:. python -m compileall -q src tests tools
PYTHONPATH=src:. pytest -q tests/rules
PYTHONPATH=src:. pytest -q
```

## Optional documentation SVG workflow

If documentation SVG files were rebuilt, apply the SVG build policy before rule
tests:

```bash
cd doc
make
cd ..

python tools/docs_svg_build_policy.py \
  --root . \
  --clean \
  --check \
  --report-file doc/maintenance/docs_svg_build_policy_report.json
```

## Operator checklist

Before using the bundle as a handoff artifact, verify:

```text
[ ] dry-run plan was reviewed
[ ] bundle was created only with --apply
[ ] manifest.json exists
[ ] read_only is true
[ ] external_call_performed is false
[ ] probe_allowed has been reviewed
[ ] pytest rules pass
[ ] full pytest suite passes
```

## Failure handling

If the CLI reports a schema mismatch, regenerate the input artifact that failed
validation.

If the CLI reports a repository mismatch, stop and inspect the three input JSON
files. A bundle must never merge artifacts from different repositories.

If rule tests fail because generated SVG files exist under
`doc/docs/architecture/context`, run the documentation SVG build policy cleanup.
