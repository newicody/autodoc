# Règle 0287 r16-r36/r16-r38 — graphe et bootstrap du laboratoire amour

- Auditer et réutiliser le constructeur existant de première visite.
- Construire un graphe typé corrélé à une seule commande GitHub.
- Conserver les deux analyses, leurs projections et leur synthèse comme tâches distinctes.
- Enregistrer les handlers explicitement au bootstrap, sans effet d’import.
- Ne cataloguer que les capacités réellement exécutables dans l’unité courante.
- Refuser un budget inférieur au nombre d’étapes ou à deux visites de spécialistes.
- Ne pas soumettre la visite, appeler un spécialiste ou publier vers GitHub dans le handler de préparation.
- Ne créer aucun Scheduler parallèle, Dispatcher applicatif, manager, daemon, file JSON ou JSONL.
- Laisser au Scheduler canonique toutes les décisions de continuation.
- Garder EventBus, PassiveSupervisor et VisPy observation-only.
